## 0. 這份文件怎麼用

`v1.0.0-rc1` 是 MR milestone（把 scene-spawn 那半 re-base 到 NVIDIA Isaac Lab `sim_utils`）完成後的 **release candidate**。本清單是「正式 tag `v1.0.0` 前的人工驗收」：

- 逐項跑「驗收方式」，比對「通過標準（目標）」，在 `[ ]` 打勾。
- 第 2–4 節全綠、且第 6 節的「刻意 scope reduction」你接受 → 第 7 節 promote `v1.0.0`。
- 任一項不過 → 記在該項下，回 MR 修，**不** promote。

驗收對象是 **base robot-sim repo 本體**（`camera_bot` 範例），**不是** forklift 應用（見第 8 節）。

## 6. 已知、刻意的 scope reduction（**不要**因這些判 fail）

驗收時這些是**設計上刻意**的，不是缺陷；C2 真人 dry-run 時請順帶判斷是否要在 `v1.0.0` 前補：

- **SimulationContext 仍 deferred**：#151 實測 `SimulationContext(...).reset()` 會 hang driver shutdown（`app.close` 不回、180s timeout）。driver/example 走 `AppLauncher` + `omni.timeline` loop；example 的 L1–L4 + cmd_vel 是結構/相機/topic 斷言，不需 physics stepping。→ **是否在 v1.0.0 前要 SimulationContext，由本次 dry-run 裁決。**
- **`prop_cube` 物理是結構性 authored**（有 `RigidBodyAPI`），但**沒有 sim stepping**，所以它不會真的掉落 / 被推。物理模擬隨 SimulationContext 一起 deferred。
- **`test_l1_urdf_to_usd_diff_zero` 已退役**：它比對 fresh import vs committed `camera_bot.usd`，後者是 legacy importer 產物（只補了 `defaultPrim`，未用新 instanceable importer 重生），diff 必非零；「fresh import 結構正確」契約已由 `test_model_import.py` 覆蓋。
- **committed `camera_bot.usd`**：是 legacy importer 輸出 + `defaultPrim` patch（usd-core，免 GPU），**非**新 importer 重生。功能正確；要 byte-level 對齊新 importer 是 out-of-band GPU 步驟。
- **lidar / imu = `NotImplementedError`**：刻意的 out-of-scope callout，已寫進 README；換 sensor 步驟釘在 in-scope（camera）。
- **Isaac Lab pin `v2.3.0`**（非最新 2.3.x）：因 `isaac-sim:5.1.0` bundle URDF importer 2.4.30、而 IsaacLab v2.3.1+ 寫死要 2.4.31（offline runner 抓不到）。詳見 `Dockerfile` 的 `ARG ISAACLAB_VERSION` 註解。
