# Changelog

## [Unreleased]

### Added
- Isaac Sim 5.1.0 base image (`nvcr.io/nvidia/isaac-sim:5.1.0`) wired in via Dockerfile `ARG BASE_IMAGE`
- `script/init_isaac_dirs.sh` — host-side first-time setup that pre-creates 8 Isaac Sim cache dirs under `${WS_PATH}/isaac-sim/` so the container's non-root user can write to them (avoids docker daemon root auto-mkdir)
- `setup.conf [environment]`: `ACCEPT_EULA=Y`, `PRIVACY_CONSENT=Y`
- `setup.conf [volumes]`: 8 cache mounts (`kit` / `ov` / `pip` / `glcache` / `computecache` / `logs` / `data` / `documents`)
- `setup.conf [deploy] gpu_capabilities`: added `video` to expose NVENC libs (`libnvidia-encode.so`, `libnvcuvid.so`) via nvidia-container-runtime. Without this the WebRTC client connects but the server cannot encode frames, producing a black viewport.
- `test/smoke/isaac_smoke.bats` — 5 image-side sanity assertions (launchers, Python 3.11, runtime user identity, isaac-sim group membership, HOME writability)
- Dockerfile `headless` + `gui` stages — non-baseline stages auto-emitted as compose services by template v0.17+ (#215). `./run.sh -t headless -d` directly launches WebRTC streaming server (entrypoint `runheadless.sh -v`); `./run.sh -t gui -d` launches X11 GUI app (entrypoint `runapp.sh`). Replaces the previous "manual `docker exec /isaac-sim/runheadless.sh` from inside `devel`" workflow.
- `setup.conf [stage:headless] gui.mode = off` — per-stage override (#220, requires template ≥ v0.18.1) that strips X11 mount + `DISPLAY` env from the headless service so kit no longer emits `X11 connection rejected because of wrong authentication` cosmetic warnings during livestream startup.
- Dockerfile `standalone` stage (`FROM devel AS standalone`, no ENTRYPOINT/CMD) — third extras stage alongside `headless` / `gui`, dedicated to the standalone Python workflow: `./run.sh -t standalone -d` brings up an idle container (base compose default `tail -f /dev/null`), `./exec.sh -t standalone /isaac-sim/python.sh <script>` runs a Python file that instantiates `SimulationApp({"livestream": 2})` and boots its own kit + WebRTC streaming server. Ctrl+C kills the script, the container stays for next exec; `./stop.sh` cleans up. Reasoning: (a) `headless` can't host this — its ENTRYPOINT already runs `runheadless.sh -v` so a second kit from `python.sh` would collide on PhysX / DDS / port 8211; (b) `devel` can't host this either — base v1's `[stage:devel]` is reserved and rejects per-stage overrides ("[setup] WARNING: [stage:devel] is reserved; not applied in v1") so we can't strip X11 from devel without breaking the rest of the devel workflow.
- `setup.conf [stage:standalone] gui.mode = off` — per-stage override mirrors `[stage:headless]`'s pattern, strips X11 mount + `DISPLAY` env from the standalone service so kit's `carb.windowing-glfw` / X11-reject cosmetic warnings stay off during livestream startup. The `gui` stage (which legitimately needs X11 for `runapp.sh`) keeps the default `[gui] mode = auto`.
- `setup.conf [environment]` ROS 2 bridge baseline (distro-agnostic) — `ROS_DOMAIN_ID=0`, `RMW_IMPLEMENTATION=rmw_fastrtps_cpp`, `FASTRTPS_DEFAULT_PROFILES_FILE=/isaac-sim/fastdds.xml`. Distro choice itself moved to `setup.conf [build] arg_N=ROS_DISTRO=...` (build-time, hard-baked — see `Distro choice` Notes entry). Aligns with NVIDIA's official [Isaac Sim 5.1 ROS 2 install guide](https://docs.isaacsim.omniverse.nvidia.com/5.1.0/installation/install_ros.html).
- `setup.conf [build] arg_4=ROS_DISTRO=humble` — distro is a build-time choice. `ARG ROS_DISTRO` in Dockerfile defaults to humble; switching = edit `setup.conf` + `./build.sh` (rebuilds only the affected layers, ~10s).
- `script/isaac-ros-env-wrapper.sh` — ENTRYPOINT shim for `headless` / `gui` stages. Reads `/etc/isaac/ros-distro` (baked at build time from `ARG ROS_DISTRO`) and unconditionally exports `ROS_DISTRO` + `LD_LIBRARY_PATH=/isaac-sim/exts/isaacsim.ros2.bridge/${ROS_DISTRO}/lib` before exec'ing the wrapped binary. Runtime `-e ROS_DISTRO=...` flags are ignored on production paths.
- `Dockerfile` devel stage `RUN echo "${ROS_DISTRO}" > /etc/isaac/ros-distro` + `ENV ROS_DISTRO=${ROS_DISTRO}` + `ENV LD_LIBRARY_PATH=/isaac-sim/exts/isaacsim.ros2.bridge/${ROS_DISTRO}/lib` — soft bake for interactive devel shell. headless / gui stages additionally route through the wrapper for hard bake.
- `test/smoke/isaac_ros_env_wrapper.bats` — 8 unit assertions on the wrapper: file presence + executability, `/etc/isaac/ros-distro` content, ROS_DISTRO + LD_LIBRARY_PATH export, **runtime override hard-ignored**, args verbatim passthrough, devel stage ENV soft bake.
- `Dockerfile` test stage now lints `script/*.sh` (ShellCheck) — covers `isaac-ros-env-wrapper.sh` + `init_isaac_dirs.sh` + future shims; previously only repo-root `*.sh` (build/run/exec/stop/setup) were linted.
- `config/ros2/fastdds.xml` — Fast DDS profile (UDPv4-only, `useBuiltinTransports=false`) lifted verbatim from the official guide. Built-in SHM transport is unreliable across containers even with `--ipc=host`; forcing UDP makes cross-container discovery deterministic on `--net=host`.
- `Dockerfile` devel stage `COPY config/ros2/fastdds.xml /isaac-sim/fastdds.xml` (mode 0644) — bakes the profile into the image so both `isaac` and any sibling `ros:humble` container can point `FASTRTPS_DEFAULT_PROFILES_FILE` at the same file (or bind-mount it to ROS 2 client containers).
- `test/smoke/docker_env.bats` — `fastdds.xml` is baked + readable + contains `<useBuiltinTransports>false</useBuiltinTransports>`.
- `test/smoke/isaac_smoke.bats` — bundled humble + jazzy `lib/librmw_fastrtps_cpp.so` + `rclpy/rclpy` are both present, image stays distro-agnostic.

### Notes
- Isaac Sim 5.1 reads privacy consent from `OMNI_ENV_PRIVACY_CONSENT`, not `PRIVACY_CONSENT`. The latter is preserved for backwards compat but no longer effective; set the former explicitly to opt in to telemetry.
- `[stage:headless] gui.mode = off` requires template **v0.18.1 or later** — v0.18.0 had a list-inheritance bug where the override was emitted on top of `extends: devel`, so X11 mounts inherited regardless. Fixed in v0.18.1 by switching to standalone-emit when any list-affecting override fires.
- ROS 2 bridge auto-loads via the default kit experience (`isaacsim.exp.full.kit` → `[settings] isaac.startup.ros_bridge_extension = "isaacsim.ros2.bridge"`); no `--enable isaacsim.ros2.bridge` launch flag is needed in `runheadless.sh` / `runapp.sh` CMD.
- Distro choice (build-time, hard-baked): default `humble` via `setup.conf [build] arg_4=ROS_DISTRO=humble`. Switch to jazzy = edit `setup.conf` + `./build.sh` rebuild (only the ENV / RUN-echo layers rebuild, not the 15 GB Isaac Sim base — ~10s). `headless` / `gui` ENTRYPOINT goes through `isaac-ros-env-wrapper.sh` which re-exports from `/etc/isaac/ros-distro` at every container start, so runtime `-e ROS_DISTRO=...` flags do not take effect. Bridge distro priority (per `isaacsim.ros2.bridge/extension.py`): `os.environ["ROS_DISTRO"]` > kit setting `exts."isaacsim.ros2.bridge".ros_distro` (resolves `system_default` → Ubuntu detection); the wrapper guarantees the env-var path wins with the baked value.
- Why a wrapper shim instead of `Dockerfile ENV` alone — Docker's `ENV` is overridable at runtime (`docker run -e ...` / compose `environment:`), which would silently mismatch with the baked `LD_LIBRARY_PATH` if a downstream user passed `-e ROS_DISTRO=jazzy`. The wrapper short-circuits that footgun by always re-reading from the build-time const file.
- Template upgraded v0.18.1 → v0.20.1: ships the `${KEY}` cross-reference fix above, plus `setup_tui` main-menu restructure (cosmetic) and an opt-in `publish-worker.yaml` reusable workflow (we don't consume it).
