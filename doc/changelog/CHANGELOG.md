### Fixed
- Dockerfile `standalone` stage: add explicit `CMD ["sleep", "infinity"]`. Without this the standalone profile-gated compose service inherited devel's `bash` CMD but with `stdin_open: false, tty: false` (base #215 profile-gated service default, vs the base devel service's `true/true`), so `bash` ran non-interactively and exited 0 immediately on `./run.sh -t standalone -d`. Symptom: `docker ps -a` showed `<user>-isaac-standalone Exited (0) <N> seconds ago`, then `./exec.sh -t standalone <cmd>` aborted with `Container '<user>-isaac-standalone' is not running` — even though `./run.sh` reported `Container <user>-isaac-standalone Started` (compose's `Started` only acknowledges the start request, not container-stays-up). `ENTRYPOINT` stays inherited from devel (`/entrypoint.sh`) — entrypoint does env init then `exec "$@"`, which now resolves to `sleep infinity` and keeps the container idle until `./stop.sh` reclaims it. Headless / gui escape this trap because they explicitly set `runheadless.sh -v` / `runapp.sh` as long-lived ENTRYPOINT/CMD.

### Notes
- Isaac Sim 5.1 reads privacy consent from `OMNI_ENV_PRIVACY_CONSENT`, not `PRIVACY_CONSENT`. The latter is preserved for backwards compat but no longer effective; set the former explicitly to opt in to telemetry.
- `[stage:headless] gui.mode = off` requires template **v0.18.1 or later** — v0.18.0 had a list-inheritance bug where the override was emitted on top of `extends: devel`, so X11 mounts inherited regardless. Fixed in v0.18.1 by switching to standalone-emit when any list-affecting override fires.
- ROS 2 bridge auto-loads via the default kit experience (`isaacsim.exp.full.kit` → `[settings] isaac.startup.ros_bridge_extension = "isaacsim.ros2.bridge"`); no `--enable isaacsim.ros2.bridge` launch flag is needed in `runheadless.sh` / `runapp.sh` CMD.
- Distro choice (build-time, hard-baked): default `humble` via `setup.conf [build] arg_4=ROS_DISTRO=humble`. Switch to jazzy = edit `setup.conf` + `./build.sh` rebuild (only the ENV / RUN-echo layers rebuild, not the 15 GB Isaac Sim base — ~10s). `headless` / `gui` ENTRYPOINT goes through `isaac-ros-env-wrapper.sh` which re-exports from `/etc/isaac/ros-distro` at every container start, so runtime `-e ROS_DISTRO=...` flags do not take effect. Bridge distro priority (per `isaacsim.ros2.bridge/extension.py`): `os.environ["ROS_DISTRO"]` > kit setting `exts."isaacsim.ros2.bridge".ros_distro` (resolves `system_default` → Ubuntu detection); the wrapper guarantees the env-var path wins with the baked value.
- Why a wrapper shim instead of `Dockerfile ENV` alone — Docker's `ENV` is overridable at runtime (`docker run -e ...` / compose `environment:`), which would silently mismatch with the baked `LD_LIBRARY_PATH` if a downstream user passed `-e ROS_DISTRO=jazzy`. The wrapper short-circuits that footgun by always re-reading from the build-time const file.
- Template upgraded v0.18.1 → v0.20.1: ships the `${KEY}` cross-reference fix above, plus `setup_tui` main-menu restructure (cosmetic) and an opt-in `publish-worker.yaml` reusable workflow (we don't consume it).
